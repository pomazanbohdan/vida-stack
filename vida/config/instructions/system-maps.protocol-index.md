## Governance / Scope Routing

1. Required gates, approval behavior, publication/contribution rules, and lifecycle policies are owned by `vida/config/instructions/system-maps.governance-map.md`.
2. Framework topology and scope boundaries are owned by `vida/root-map.md` and `vida/config/instructions/system-maps.framework-map-protocol.md`.
3. Concrete runtime command paths are owned by runtime-family maps and their runtime homes, not by this registry.

## Maintenance Rule

When a protocol changes:

1. Update the canonical file first.
2. Update linked references in the same change.
3. Keep this index synchronized.
