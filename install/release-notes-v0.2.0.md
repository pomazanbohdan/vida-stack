# 🌌 Vida Stack v0.2.0

## ✨ Highlights

- 🚀 First installable `VIDA 0.2.0` release path with versioned archive packaging and local installer flow.
- 🧭 Canonical root-map architecture for framework, project, runtime-family, template, and governance discovery.
- 🧱 Green runtime-layer matrix for `taskflow-v0`, including runtime kernel bundle, role selection, verification merge, checkpoint recovery, and final `taskflow -> codex` consumption.
- 📚 Stronger documentation authority through `codex-v0`, canonical layer maps, readiness checks, and proof surfaces.
- 👥 Expanded role system groundwork with `business_analyst`, `pm`, `coach`, `verifier`, `prover`, shared skills, and project extensions.
- Party Chat integration over the legacy `problem_party` slot, with project-owned council roles/profiles/flows and config-driven role-to-model bindings.

## 🛠️ Release Assets

- source archive with framework maps, docs, runtime, installer, and scripts
- Linux `taskflow-v0` binary staged inside the release archive
- local installer for versioned install, upgrade, shell wiring, and wrapper creation
- checksums for archive verification

## ✅ What This Release Makes Practical

- install `VIDA 0.2.0` locally through one archive-driven flow
- run `taskflow-v0` from a stable installed root
- run `codex-v0` through an installer-managed Python environment
- keep multiple installed versions and switch the active `current` release safely
- work from canonical framework/project/runtime maps instead of ad hoc bootstrap paths
- render Party Chat manifests and receipts with project-aware facilitator/expert bindings

## 🔭 Direction

`v0.2.0` is the proving release that stabilizes the transitional runtime, canonical maps, and installer path while preserving the separation needed for `VIDA 1.0`:

- `taskflow` owns runtime execution
- `codex` owns canonical documentation and inventory validation
- future `vida` composes them without collapsing their boundaries
