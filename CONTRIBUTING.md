# Contributing

Purpose: define the current contribution posture and collaboration boundaries for the repository.

Vida Stack is still in active architectural formation.

## Current Contribution Policy

At this stage, external code contributions are **not currently accepted**.

What is welcome:

1. issue reports
2. bug reports with reproduction details
3. design feedback
4. roadmap discussion
5. specification review comments

What is not currently accepted:

1. pull requests with code changes
2. pull requests with framework behavior changes
3. pull requests that modify core protocols, runtime rules, or product direction

## Why

The framework core, binary roadmap, and licensing/contribution model are still being stabilized.

Until that work is complete, the project is intentionally keeping authorship and rights management simple.

## Future Plan

When the project matures enough to accept external code contributions, Vida Stack intends to introduce:

1. a formal contribution policy,
2. contributor license agreement handling,
3. a clearer distinction between open core and future commercial/proprietary layers.

Until then, please use issues and discussions for collaboration.
